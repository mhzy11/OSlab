# Lab1 中断处理流程

## 1. 中断注册流程

```mermaid
sequenceDiagram
    participant K as kern_init()
    participant T as trap_init()
    participant H as hmcode (vcpucb)
    participant C as clock_init()
    participant I as intr_enable()

    K->>T: 调用
    T->>H: wrkgp(gp) [HMC 0x37]
    Note over H: 保存内核 GP
    T->>H: wrent(entInt, 0) [HMC 0x34]
    Note over H: 保存中断入口 entInt
    T->>H: wrent(entIF, 3) [HMC 0x34]
    Note over H: 保存异常入口 entIF
    T-->>K: 返回

    K->>C: 调用
    C->>H: wrtimer(1) [HMC 0x3B]
    Note over H: 启动 oneshot 定时器
    C-->>K: 返回

    K->>I: 调用
    I->>H: swpipl(0) [HMC 0x35]
    Note over H: CPU 开中断
    I-->>K: 返回

    Note over K: while(1) 等待中断
```

## 2. 中断发生与处理流程

```mermaid
sequenceDiagram
    participant T as 定时器 (硬件)
    participant H as hmcode
    participant E as entInt (汇编)
    participant D as do_entInt (C)
    participant R as __trapret

    T->>H: 定时器到期，硬件中断

    Note over H: host_interrupt1.S
    H->>H: 读 CSR__INT_STAT
    H->>H: 判断为定时器中断
    H->>H: r16 = 9 (OSF_A0_INT__CLK)
    H->>H: 保存 PS/PC/GP/r16/r17/r18 到栈
    H->>H: 从 vcpucb 读 kgp
    H->>H: 从 vcpucb 读 entInt

    H->>E: 跳转 entInt (带 kgp)

    Note over E: SAVE_ALL
    E->>E: 保存 r0~r28 到栈
    E->>E: cause = 0
    E->>E: sp 作为参数 (trapframe*)
    E->>D: call trap() → do_entInt()

    D->>D: wrtimer(1) 重新触发
    D->>D: ticks++
    D->>D: 打印 (每 100 次)

    D->>R: 返回

    Note over R: RESTORE_ALL
    R->>R: 恢复 r0~r28
    R->>H: HMC_rti [HMC 0x3F]

    Note over H: 从栈恢复 PS/PC/GP/r16-r18
    H->>T: 返回被中断的代码 (while(1))
```

## 3. 异常处理流程

```mermaid
flowchart TD
    subgraph 断点异常
        A1["sys_call 0x80 (HMC_bpt)"] --> B1["hmcode 识别"]
        B1 --> C1["r16 = 0 (IF_BREAKPOINT)"]
        C1 --> D1["entIF"]
    end

    subgraph 非法指令异常
        A2[".long 0x7a000000"] --> B2["CPU 解码失败"]
        B2 --> C2["QEMU: EXCP_OPCDEC"]
        C2 --> D2["hmcode opcdec.S"]
        D2 --> E2["r16 = 4 (IF_OPDEC)"]
        E2 --> D1
    end

    D1 --> F["SAVE_ALL (r0~r28)"]
    F --> G["cause = 3"]
    G --> H["trap(tf)"]
    H --> I{cause == 0?}
    I -->|否| J["do_entIF(tf)"]
    J --> K{a0 (r16) = ?}

    K -->|0| L["断点<br/>打印 PC-4<br/>不需要 +4"]
    K -->|4| M["非法指令<br/>打印 PC<br/>PC += 4 跳过"]

    L --> N["__trapret"]
    M --> N
    N --> O["RESTORE_ALL"]
    O --> P["HMC_rti (0x3F)"]
    P --> Q["返回执行"]
```

## 5. 整体架构

```mermaid
flowchart LR
    subgraph 内核
        direction TB
        KERNEL["kern_init()"]
        TRAP["trap_init()"]
        CLOCK["clock_init()"]
        INTR["intr_enable()"]
        ENTRY["entInt / entIF<br/>(汇编入口)"]
        HANDLER["trap() → do_entInt/do_entIF<br/>(C 分发)"]
        RET["__trapret<br/>RESTORE_ALL + HMC_rti"]
    end

    subgraph hmcode
        HM["host_interrupt1.S<br/>opcdec.S"]
        VCPUCB["vcpucb<br/>(内核入口表)"]
    end

    subgraph 硬件
        CPU["SW64 CPU"]
        TIMER["定时器"]
    end

    KERNEL --> TRAP -->|wrkgp + wrent| VCPUCB
    KERNEL --> CLOCK -->|wrtimer| TIMER
    KERNEL --> INTR -->|swpipl| CPU
    
    TIMER -->|中断| CPU
    CPU -->|EXCP| HM
    HM -->|读入口| VCPUCB
    HM -->|跳转| ENTRY
    ENTRY --> HANDLER
    HANDLER --> RET
    RET -->|HMC_rti| HM
    HM -->|返回| CPU
```
